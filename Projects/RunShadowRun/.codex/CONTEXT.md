# CONTEXT — RunShadowRun

## Structure repo
- src/        : code
- assets/     : assets (placeholder puis finals)
- tools/      : scripts utilitaires (build, logs, etc.)
- .codex/     : mémoire projet

## Conventions
- Nommage: clair, stable, pas d'abréviations obscures
- Petits commits fréquents (si Git utilisé)
- Chaque session: mise à jour JOURNAL + TODO si nécessaire

## Notes
- Le choix exact du moteur/stack peut être décidé au démarrage (checkpoint "TECH DECISION").
- On vise d'abord jouable avec placeholders, puis habillage.

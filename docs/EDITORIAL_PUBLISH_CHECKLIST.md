# Editorial publish checklist

Checklist simple avant de considerer un lot de contenu comme propre a publier ou a mettre fortement en avant.

## 1. Structure minimale

- le terme a un `slug` propre
- la categorie est correcte
- la sous-categorie est correcte si necessaire
- le statut final est coherent

## 2. Qualite de lecture

- la definition est claire
- la definition se termine proprement
- l’exemple est utile et concret
- le ton reste explicatif
- les accents et la ponctuation sont corrects

## 3. Valeur editoriale

- la fiche apporte vraiment quelque chose
- le terme n’est pas un doublon deguisé
- les termes lies sont utiles
- la fiche aide un debutant, pas seulement un expert

## 4. Cohérence metier

- le vocabulaire correspond bien au contexte architecture / chantier / bureau
- les normes mentionnees sont pertinentes
- les applications et points de vigilance sont credibles
- la fiche ne melange pas plusieurs notions sans raison

## 5. Medias

- le media est utile, pas decoratif seulement
- le format est bon
- le lien fonctionne
- le media renforce vraiment la comprehension

## 6. Equilibre du corpus

- la categorie n’est pas oubliee par rapport aux autres
- la fiche s’insere bien dans un ensemble de lecture
- le lot publie ne cree pas une incoherence forte entre categories

## 7. Verification avant publication

- `npm run content:validate`
- `npm run content:lint:fr`
- `npm run content:stats`

## 8. Question finale

Si un apprenti ou un debutant ouvre cette fiche, peut-il:

- comprendre le mot
- le replacer dans son contexte
- retenir l’essentiel
- ouvrir ensuite une autre fiche utile

Si la reponse n’est pas clairement oui, la fiche doit encore etre retravaillee.
